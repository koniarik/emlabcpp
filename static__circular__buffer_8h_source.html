222  //
223  // Set of [delete,init,emplace]_item methods is necessary as data_ is not array of T, but
224  // array of byte-like-type that can store T -> T does not have to be initialized there. We
225  // want to fully support T objects - their constructors/destructors are correctly called and
226  // we do not require default constructor. This implies that data_ has some slots
227  // un-initialized, some are initialized and we have to handle them correctly.
