In both methods, we create the bounded size without properly checking that it the bounded type was made properly (that is, that the provided std::size_t value is in the range).

Thas is ok as long as variant "we advanced the iter only by at max `sub_converter::max_size` `N` times".

References EMLABCPP_ASSERT, and emlabcpp::bounded< T, MinVal, MaxVal >::make().
