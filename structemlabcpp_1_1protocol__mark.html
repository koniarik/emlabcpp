Detailed Description

Error handling in protocol works with marks.

Mark is a string made of eight characters that represents label. Each error is labeled by namespace specific for the source project and by name unique in that namespace, part of the error is also index of byte that caused the problem. These can be easily sent with the protocol lib itself.
